# 综合化工桌面管理系统

题目 10 对应的小型桌面管理系统，采用 Python 标准库 `tkinter` 开发，无需第三方依赖。

## 运行方式

```bash
python main.py
```

也可以在 PyCharm 中直接运行 `main.py`。

## 默认账号

| 角色 | 账号 | 密码 | 权限说明 |
| --- | --- | --- | --- |
| 管理员 | `admin` | `admin123` | 可调整库存、确认报警、配置阈值、删除/关闭工单、导出报表 |
| 普通操作员 | `operator` | `operator123` | 可查询数据、查看监控、新建/更新工单、预览报表 |

## 功能模块

- 权限管理：管理员可维护用户、角色和角色权限，权限配置存储在 SQLite 表中。
- 数据存储：业务数据由 `data/cms_data.db` SQLite 数据库管理，首次启动会根据默认数据自动初始化。
- 数据查询：按关键字、类别、危险等级查询化学品台账，管理员可进行入库/出库调整。
- 参数监控：支持自动定时采样、实时趋势图、报警弹窗、报警历史，管理员可确认报警和修改阈值。
- 工单管理：新建、编辑、状态流转和删除工单，普通操作员不能删除或关闭工单。
- 报表导出：支持综合简报、化学品库存、参数监控、工单台账 CSV 导出，导出文件位于 `reports/`。
- 运行看板：登录后默认展示库存风险、报警数量、未关闭工单和关键列表。
- 操作审计：记录入库/出库、报警确认、阈值修改、工单变更、报表导出等关键操作，管理员可在看板查看。
- 全局异常处理：统一弹窗提示用户可理解的错误，并把异常记录到 `logs/app.log`。

## 工程结构

```text
app/
  auth.py            # 身份认证与权限逻辑
  database.py        # SQLite 建表与初始化
  data_store.py      # SQLite 数据仓库
  exceptions.py      # 全局异常与日志
  report_service.py  # 报表预览与导出
  seed_data.py       # 默认数据与权限清单
  ui/                # Tkinter 界面模块
data/cms_data.json   # 初始化种子数据
data/cms_data.db     # 运行时 SQLite 数据库（首次启动自动生成）
main.py              # 程序入口
tests/               # 核心逻辑测试
```
